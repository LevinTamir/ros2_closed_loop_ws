# ROS2 Closed Loop Kinematics Workspace

A test bed for working with closed-loop kinematics in Gazebo using ROS2.

This workspace contains the `delta_arms_description` package, which demonstrates modeling and
simulating robots with closed kinematic chains in the new Gazebo simulator. It includes three
examples:

- **volcaniarm** — a planar 5-bar linkage.
- **3-DoF delta** — a translational delta parallel robot (ported from `PARA_ENGINEER`).
- **4-DoF delta** — a delta with a central telescopic shaft giving the end-effector an extra
  rotation axis (ported from Autonox `A_00036`).

## How the closed loops are modeled

URDF can only describe open kinematic trees, so the loops are closed at runtime with Gazebo's
`DetachableJoint` system plugin: a massless tip link on one chain is welded to a coincident dummy
link on the other. The frames are made coincident at the assembled "home" pose (baked into the
joint origins) so the weld attaches with zero impulse. Actuated joints are driven by
`JointPositionController` plugins over `/<robot>/<joint>/cmd_pos` (`std_msgs/Float64`) topics, and
all joint states are published by a `JointStatePublisher` plugin.

For the deltas, each leg's forearm is a spherical-jointed rod (`weld + 3R` chain == a ball joint,
matching the source `spherical_joint` constraints), and the moving platform hangs from a passive
3-prismatic X-Y-Z chain so it keeps exactly 3 translational DOF and stays level by construction.
The delta URDFs are generated by the scripts in `delta_arms_description/scripts/`.

## Requirements

- ROS2 Jazzy (Tested)
- Gazebo Harmonic

## Installation

Clone the repository:

```bash
git clone git@github.com:LevinTamir/ros2_closed_loop_ws.git
cd ros2_closed_loop_ws
```

Build the workspace:

```bash
colcon build
```

Source the workspace:

```bash
source install/setup.bash
```

## Usage

Launch any of the examples:

```bash
# planar 5-bar linkage
ros2 launch delta_arms_description launch_volcaniarm.launch.py

# 3-DoF translational delta
ros2 launch delta_arms_description launch_delta_3dof.launch.py

# 4-DoF delta (with telescopic shaft + rotating end-effector)
ros2 launch delta_arms_description launch_delta_4dof.launch.py
```

Command the actuated joints from another terminal (positions in radians). For the 3-DoF delta,
equal angles on all three arms move the platform straight up/down, unequal angles move it sideways
— the platform always stays level:

```bash
ros2 topic pub -1 /deltaarm_3dof/Chain1_1/cmd_pos std_msgs/msg/Float64 "{data: 0.4}"
ros2 topic pub -1 /deltaarm_3dof/Chain2_1/cmd_pos std_msgs/msg/Float64 "{data: 0.4}"
ros2 topic pub -1 /deltaarm_3dof/Chain3_1/cmd_pos std_msgs/msg/Float64 "{data: 0.4}"
```

The 4-DoF delta adds `Chain4_1`, which rotates the end-effector flange:

```bash
ros2 topic pub -1 /deltaarm_4dof/Chain4_1/cmd_pos std_msgs/msg/Float64 "{data: 1.0}"
```
